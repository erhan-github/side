## 2. Level 4: Feedback-Driven Intelligence (Phase 4)
*Goal: The system gets smarter based on HOW you ignore it.*

### 2.1 The Implicit Signal Ledger
We track every suggestion rendered in the Monolith.
- **Accepted**: If a finding is resolved within 2 hours of being viewed.
- **Ignored**: If a finding persists for 7+ days.
- **Rejected**: If the user adds a `// side:ignore` tag.

### 2.2 Dynamic Weight Recalibration
Side automatically adjusts its `DIMENSION_WEIGHTS` (Security, Performance, etc.) based on user focus. If you always fix Performance first, Side escalates Performance findings to TOP PRIORITY.

## 3. Level 5: Autonomous Self-Healing Forensics (Phase 5)
*Goal: Side fixes the codebase before you notice the "Broken Window."*

### 3.1 The Remediation Loop
1.  **Detect**: ForensicEngine finds a violation (e.g., N+1 Query).
2.  **Verify**: Cross-references with @Strategic OCP tags.
3.  **Propose**: Side generates a branch/PR with the fix + regression test.
4.  **Validate**: Side runs the tests autonomously in a sandbox.

### 3.2 Recursive Error Correction
If a fix breaks the build, Side analyzes the CI log and recurses until the fix is "Safe to Auto-Run."
